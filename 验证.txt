SMoPE CIFAR-100 十任务遗忘来源分析

分析日期：2026-07-16  
数据目录：`outputscifar-10010-task`  
主要依据：Accuracy Matrix、组件冻结、路由回放、参数漂移、梯度方向、专家使用和组件恢复。

## 一、先说结论

当前实验没有确认某一个单独组件是完整、唯一的遗忘根因。这是一个可靠的否定性结论，不是“实验白做了”：按照 runbook 的严格标准，一个组件必须同时满足“确实变化、更新方向伤害旧任务、恢复历史状态能救回旧任务、冻结后能提高旧任务保留且不明显伤害新任务”才能被称为重要遗忘来源。现在没有任何单组件同时满足全部条件。

证据支持以下分级判断：

1. Key 与 Value 的联合状态漂移是最强的功能性嫌疑对象，但还不能称为已证实根因。恢复历史 Key+Value 平均找回 `2.067` 个准确率百分点，2727 个 seed×旧任务配对均为正；对原本确实发生遗忘的配对，合计找回约 `46.8%` 的遗忘缺口。联合恢复比 Key、Value 单独恢复之和还多约 `0.400` 点，说明两者存在协同作用。但单独冻结 Key 的收益很小且跨 seed 不一致，冻结 Value 反而明显变差；KeyValue 的实际更新方向平均也没有增加旧任务损失。因此能说“最终 Key+Value 状态中含有大量可恢复的遗忘信息”，不能说“训练时更新 Key+Value 就是遗忘根因”。
2. 路由的软强度（Prompt logits）是次要但较稳定的遗忘贡献因素；硬专家身份变化只占较小部分。最终 checkpoint 相比历史 checkpoint，旧任务硬路由平均有 `19.97%` 的位置发生变化。只恢复历史专家身份平均找回 `0.159` 点；再恢复历史 Prompt logits 后总共找回 `0.563` 点，2127 个配对为正，约覆盖正遗忘缺口的 `14.2%`。因此路由确实参与遗忘，但不是主要解释；多数可恢复收益来自软注意力强度，而非“选错了哪个专家”。
3. Key 单独是弱候选，证据不足。冻结 Key 后最终旧任务准确率平均只提高 `0.078` 点，而且只有 13 个 seed 提高；FAA 平均仅提高 `0.030` 点。这个量级小于 seed 间波动，不能据此确认 Key 更新导致遗忘。
4. Value 单独存在明显的最终状态恢复能力，但整体更新更像学习所必需，而不是已证实的遗忘源。恢复历史 Value 平均提高 `1.041` 点，但冻结 Value 使最终旧任务准确率下降 `0.859` 点，新任务可塑性损失 `1.759` 点。也就是说，历史 Value 能救回旧任务，不等于训练中不该更新 Value。
5. 当前分类器状态不支持“分类器是遗忘来源”，反而更像补偿保护因素。冻结分类器使最终旧任务准确率在 33 个 seed 中都下降，平均下降 `1.022` 点，同时 FR 反而增加 `0.822` 点；恢复历史分类器平均再降低 `1.956` 点，2627 个配对为负。分类器校正阶段虽然有不少单步梯度会局部伤害旧任务，但长期训练后的新分类器整体优于历史分类器。
6. 把整个 Prompt 子系统冻结不能保护旧知识。冻结 Prompt 后最终旧任务准确率 33 个 seed 均下降，平均下降 `1.067` 点，新任务可塑性损失 `2.215` 点。它的 FR 虽然下降，但这是因为新任务学得更差、对角线基准下降，不能解释为真正保护了旧任务。

一句话概括：当前最合理的科研表述是“遗忘主要表现为共享 KeyValue 联合状态与软路由强度的累积失配，其中 Key+Value 联合失配包含最多可恢复信息；但现有冻结和梯度证据尚未证明这些参数的正常更新是遗忘的直接根因。分类器更像补偿者而不是破坏者。”

## 二、这些数据能否直接比较

五组主要实验的核心设置匹配：

- 10 个任务；
- `CRCT_EPOCHS=50`；
- 3 个配对 seed：0、1、2；
- 相同模型、任务顺序、优化设置和 `prompt_param`；
- baseline、freeze-prompt、freeze-key、freeze-value、freeze-classifier 的样本 manifest SHA-256 完全相同：`7e574c8e84759c4fb76079d0f8571f922225ab3a6b2f36e14a7f23f56f07ac97`。

所以当前的条件差异可以做配对描述。但是只有 3 个 seed，少于 runbook 建议的 5 个 seed，因此下文只能说“稳定趋势”“支持不支持”，不能写“统计显著证明”。

## 三、新手先理解四个指标

- FAA：训练完全部任务后的平均准确率。越高通常越好。
- CAA：整个持续学习过程中的累计平均表现。越高通常越好。
- FR：任务从历史最好表现到最终表现的下降量。通常越低越好，但必须同时看新任务是否学会。
- 最终旧任务准确率：最后模型在任务 1～9 上的平均准确率。这是判断是否真正保住旧知识的关键指标。
- 可塑性损失：冻结条件相对 baseline 在“刚学完当前新任务时”的准确率损失。数值越大，说明新任务没有正常学会。

最常见的误区是只看 FR。假设某条件让新任务从一开始就只学到 80 分，最后仍是 78 分，FR 只有 2；baseline 先学到 95 分，最后是 90 分，FR 是 5。前者 FR 更小，却不能说记忆更好，因为最终准确率 78 仍远低于 90。

## 四、冻结实验：是否“禁止更新就更不容易忘”

以下均为相对同 seed baseline 的平均差值，单位是准确率百分点：

 冻结条件  ΔFAA  ΔCAA  ΔFR  Δ最终旧任务准确率  新任务可塑性损失  判读 
---------------------
 Prompt 全部  -1.350  -0.896  -0.719  -1.067  2.215  FR 变小是假象，旧任务和新任务都更差 
 Key  +0.030  -0.100  -0.296  +0.078  0.281  近乎中性，收益仅出现在 13 seed 
 Value  -1.033  -0.729  -0.619  -0.859  1.759  更新 Value 整体上是必要的 
 Classifier  -0.883  -0.553  +0.822  -1.022  0.044  冻结后遗忘更多，当前分类器具有保护补偿作用 

详细 seed 一致性：

- Prompt 的最终旧任务差值为 `-1.078-1.089-1.033`，33 都变差。
- Key 为 `-0.100-0.044+0.378`，只有 seed 2 变好，方向不一致。
- Value 为 `-1.033-0.733-0.811`，33 都变差。
- Classifier 为 `-0.989-1.022-1.056`，33 都变差。

因此，冻结实验只给 Key 留下了一个很弱的可能性；Prompt、Value、Classifier 均不支持“停止更新即可保护旧知识”。

## 五、路由回放：遗忘来自选错专家，还是专家权重变了

在最终 checkpoint 10 上评估任务 1～9，共有 27 个 seed×旧任务配对：

 干预  平均恢复增益  正增益配对  对正遗忘缺口的合计恢复比例  含义 
---------------
 只恢复历史专家身份  +0.159  1827  3.5%  硬路由身份贡献较小 
 身份 + 历史 Prompt logits  +0.563  2127  14.2%  软路由强度贡献更明显 

其中额外恢复 Prompt logits 本身平均再增加 `0.404` 点，明显大于只恢复专家身份的 `0.159` 点。

路由变化还呈现明显的任务年龄效应：任务 1 的硬路由变化率平均为 `34.72%`，而任务 9 为 `9.73%`。任务 1 的平均遗忘缺口为 `10.467` 点，完整路由回放平均找回 `1.967` 点。说明越早的任务积累了越多路由失配，但路由仍只能解释其中一部分。

结论：路由参与遗忘，但“硬选错专家”不是主因；软注意力Prompt logits 的变化更值得关注。

## 六、组件恢复：最终模型里哪部分保存了可恢复信息

把最终模型的某个组件替换成对应旧任务 checkpoint 的历史组件：

 恢复条件  平均恢复增益  正负增益配对  对正遗忘缺口的合计恢复比例  判读 
---------------
 Router identity  +0.159  18 正  5 负  3.5%  小贡献 
 Router identity + logits  +0.563  21 正  6 负  14.2%  次要但较稳定 
 Key  +0.626  19 正  8 负  15.7%  单独有部分恢复能力 
 Value  +1.041  21 正  5 负  23.7%  单独恢复能力较强 
 Key + Value  +2.067  27 正  0 负  46.8%  最强候选，且存在协同 
 Classifier  -1.956  1 正  26 负  -41.5%  历史分类器通常更差 
 完整历史 checkpoint  +4.070  与平均遗忘缺口一致  100%  正对照通过，恢复流程可信 

Key 单独 `+0.626`，Value 单独 `+1.041`，两者相加为 `1.667`；联合恢复是 `2.067`，多出的 `0.400` 是正交互项。这说明 Key 与 Value 不是两个完全独立的遗忘通道，而是共同决定“取哪个专家、取出什么内容”的耦合系统。

最早三个任务的证据尤其明显：

 任务  平均遗忘缺口  路由身份+logits恢复  Key+Value恢复 
------------
 1  10.467  1.967  4.633 
 2  8.067  0.467  3.633 
 3  7.267  0.633  3.233 

但“恢复有效”只能证明最终状态失配与旧任务性能有关，不能单独证明训练更新是最初造成遗忘的原因。历史组件可能相当于一个有利于旧任务的干预，而不是训练过程中本来就应该完全冻结的组件。

## 七、梯度方向：训练的真实更新是否立即伤害旧任务

原始 `gradient_direction.jsonl` 实际有 540 条记录。对每个组件，135 条记录对应 3 个 seed、任务 2～10 与其所有旧任务配对。

`first_order_old_loss_change  0` 表示这一步真实 AdamW 更新在一阶近似下会增加旧任务损失，即局部有害；小于 0 表示局部有利。

 阶段组件  平均一阶旧损失变化  有害配对数  判读 
------------
 主 Prompt 训练  Key  -0.007081  49135  平均不是有害方向 
 主 Prompt 训练  Value  -0.008755  50135  平均不是有害方向 
 分类器校正  Classifier  +0.004925  95135  多数单步存在局部冲突 

Key 的 seed 0 平均略为有害 `+0.001978`，但 seed 1、2 分别为 `-0.011513-0.011709`；Value 也只有 seed 0 略正，另两个 seed 为负。这与“KeyValue 更新普遍直接伤害旧任务”的说法不一致。

分类器校正有 95135 个局部有害方向，但冻结和恢复实验表明训练后的新分类器整体更好。这说明单步局部冲突在后续训练中可能被补偿，或分类器必须牺牲一部分旧类别边界来建立更合理的全类别边界。梯度方向用于发现冲突，不能越过最终性能干预直接判定根因。

## 八、参数漂移：变化大是否等于遗忘来源

最终 checkpoint 相对旧任务 checkpoint 的平均相对 L2 漂移：

 组件  平均相对 L2  与遗忘缺口的原始相关  去除任务年龄平均趋势后的相关 
------------
 Key  0.505  0.925  0.148 
 Value  0.496  0.927  0.326 
 Classifier  0.197  0.777  -0.055 

原始相关很高，但主要原因是“任务越早，经历的后续训练越多，所以漂移更大；同时早期任务也更容易忘”。去掉每个任务的平均年龄趋势后，相关明显下降。因此漂移是一个定位线索，不能作为因果证据。Value 的去趋势相关 `0.326` 比 Key 更高，和 Value 恢复增益较大相互呼应，但仍不足以单独定因。

## 九、共享专家结构意味着什么

模型使用全局共享专家池，没有任务私有专家：6 个 Prompt 层、每层 12 个头、每头 25 个专家、每次激活 top-5，即单次激活比例 20%。

最终 baseline 中活跃专家的任务归属熵非常高：三个 seed 的平均归一化熵约为 `0.9970.9910.993`，使用量加权后都接近 `1.0`；最大任务份额加权后约为 `0.10`。直观地说，同一批活跃专家几乎被十个任务平均共享，而不是一个专家只服务一个任务。

这提供了合理机制解释：新任务更新共享 KeyValue 时，旧任务也会读到这些被改过的专家，因此具备发生干扰的结构条件。但“共享”只是易感性，不等于已经证明共享参数更新造成遗忘。

## 十、为什么几条证据会互相矛盾

这里最容易困惑的是：历史 KeyValue 恢复有效，为什么冻结它们却没有稳定收益？原因可能同时包括：

1. 稳定性与可塑性的矛盾。KeyValue 必须变化才能学新任务；完全冻结会保护一部分旧状态，也会让新任务学不好，最终不一定更优。
2. 恢复是终点干预，冻结是全程干预。恢复只在训练结束后把旧状态装回来，不承担学习新任务的代价；冻结从任务 2 开始一直限制训练，问题更难。
3. Key 与 Value 有协同。只冻结或恢复一个组件，可能被另一个组件的变化抵消。当前已有联合恢复，却还没有联合冻结对照。
4. 一阶梯度只描述局部。单步更新对旧任务有利，不保证许多步累计后的非线性路径仍有利；反之亦然。
5. 任务年龄是混杂因素。早期任务既漂移更大，也经历更多后续任务，不能把两者相关直接写成因果。

## 十二、下一步最值得做的实验

优先级从高到低：

1. 补做 Key+Value 联合冻结。这是当前最关键的缺口，因为联合恢复最强且有正交互，但代码实验目前只有单独 freeze-key 和 freeze-value。必须继续使用同一 manifest、同一 seed、同一任务顺序。
2. 把配对 seed 增加到至少 5 个。尤其是 baseline、freeze-key、未来 freeze-key+value。Key 当前只有 13 seed 提高，3 个 seed 无法判断它是小真实效应还是随机波动。
3. 做 matched-plasticity 对照。对 PromptValue 冻结，应构造一个 baseline 对照，使新任务对角线准确率下降到相近水平，再比较最终旧任务准确率。否则无法排除 FR 下降只是“没学会新任务”。
4. 做部分保护而不是完全冻结。例如只约束旧任务高频专家、只保护 top 90% 使用量的专家，或对 Key+Value 使用较小学习率梯度投影。完全冻结可能过强，掩盖了稳定性收益。
5. 重点报告任务 1～3。这些任务的遗忘缺口与 Key+Value 恢复都最大，最适合做样本级路由、专家和类别错误分析。
6. 分别处理分类器“局部梯度冲突”和“最终保护作用”。可以测试只投影削弱分类器校正中对旧任务有害的梯度，而不是冻结分类器；当前证据明确表明直接冻结不是好方案。

如果联合冻结仍不能稳定提高最终旧任务准确率，且 matched-plasticity 后也无收益，那么更可靠的结论将是：Key+Value 历史状态是一种有效的事后恢复记忆，但正常 Key+Value 更新本身不是主要遗忘根因；遗忘更可能来自多个共享组件的路径依赖和联合失配。

## 十三、数据与自动报告中的注意事项

- `forgetting-audit-summaryforgetting_audit_report.md` 写着 `Gradient-direction records 0`，但原始文件 `forgetting-audit-gradientforgetting_auditgradient_direction.jsonl` 实际有 540 条记录。原因是当前统一汇总没有把独立 gradient run 纳入该计数。本报告直接读取了原始 540 条记录。
- 组件恢复原始文件有 567 条记录，位于 `forgetting-audit-routerforgetting_auditcomponent_restoration.jsonl`；自动报告只给了通用说明，没有汇总这些数值，本报告进行了独立聚合。
- 所有“恢复比例”只对遗忘缺口大于 0 的配对合计，避免任务出现正向迁移时分母为负而产生误导。
- 结果单位均为准确率百分点，不是 0～1 比例。
